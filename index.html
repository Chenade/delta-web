Product Innovativeness:

Due to the high car accident rate in the society, we came up with a new car recorder having multiple functions. This product has the following two main usage scenarios:

  1. General situations
    1. The AI recognition will track the vehicle in front of the user and the lane which the user is currently on. The device will alert when the driver encounters some unexpected situations, the unusual speed approaching the car ahead and the user shifting outside the lane unconsciously.
    2. Especially for motorcycle riders, if there is a bus or truck detected within sight, the AI recognition will compute the inner wheel difference range based on the outer volume. Then, if the rider was inside or near the danger area, the rider will be alerted with the buzzer.
    3. With the GPS detector, the device would receive the data about the speed limitation for the current road, police speed traps, construction-related events and accident information, which was reported to our network platform, within a certain distance.
    4. Before getting on the road, users can simply go to our network platform, check for the pre-schedule construction event and take another route to avoid a traffic jam. In addition, users can set up their own route, which can be used to send notifications automatically whenever there is an event on the route.
  2. Accident situation
    1. At the scene

      2. If a user is involved in a car accident, the user can press the SOS button on the device which will report the GPS location to the network platform and the police; then the current video will be marked at the cloud at the same time. Before the accident is solved, every device within a certain distance or passing by will receive an emergency notification and alert the user to be aware.

    2. After the accident

      3. When the user who had a car accident needs these videos for several reasons, such as accident evidence, insurance, etc. Users can go to our network platform and fill in the form to ask the owner for their records; if the owner agrees, then the user can get the videos which are muted according to the privacy issue.

With this new-concept car recorder, we can effectively decrease the car accident rate and make drivers, especially scooter drivers, be more safe.

Technology readiness:

The technical applications of this product:

With 5G technology as the core foundation, this work is done by combining the characteristics of 5G high speed, low latency and wide connectivity with cloud computing and machine learning.

  1. Media Streaming

    2. The images taken by the car recorder are sent to the cloud through the socket server and processed by the advantage of 5G for real-time image recognition; if there is danger detected, the information would be sent back to the user's car recorder device immediately. Meanwhile, the video will be automatically saved on the cloud, which can reduce not only the expense on the SD Card, but the chance the video will be found out broken when needed. Additionally, users can login to our either website or app and get the video and gps information for the device.

  2. Image Recognition

    3. We use cloud computing for image recognition, and it is also built on the basis of 5G technology; in this way, we can reduce hardware requirements for the car recorder. Besides, we hope that this product can do vehicle recognition, speed analysis, lane-lines finding, inner wheel difference calculation, traffic light recognition and speed limit recognition. Image recognition is mainly performed by OpenCV, deep learning, Pytorch and YOLO(You only look once) algorithm. We try to train our own yolov3 model based on Pytorch and open source datasets. By implementing YOLO, we can dramatically increase our calculating speed and accuracy. We also implement some image processing algorithms based on OpenCV to achieve our goals.

  3. Big data analysis

    4. When the server receives the request of a video application for assistance, Server will soon filter the nearby video and the date, and process each gps record with cloud computing in order to find out the specific video that might help the applicant. Next, the server will send an email to each owner of a potential help video, which includes the basic video information and the video links, and the owner will login to our network platform whether to agree or decline.

Plans for sustainable development:

The main development of this product can be divided into the following stages:

  1. Currently completed

    2. We use Arduino Yun Rev 2 attached with a webcam, a gps detector, and a buzzer as a prototype of an end-user device. Currently, we could stream the scene within the intranet, and the image recognition will process with the intranet url. The device will send a GET HTTP Request to the server to ask for the event details nearby and update the device location every 3 seconds. For image recognition, now we can approximately find the lane-lines, and distinguish some kinds of vehicles such as bus, truck, car and motorbike, and we also can calculate the speed of approaching within two cars. For the network platform, we use node.js to develop REST API for communication, and AWS cloud service to host it. Detail can be view with following url:
